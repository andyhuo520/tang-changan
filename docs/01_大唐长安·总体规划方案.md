# 大唐长安·数字孪生总体规划方案

> Codename: **Project Chang'an** — A scalable, data-driven 3D restoration of Tang Dynasty 长安城 (开元盛世 c.742 AD)
>
> 文档版本: v1.0 (规划)  ·  目标受众: 立项 / 设计 / 开发 / 文旅推广

---

## 0. 摘要 (TL;DR)

| 维度 | 现状 (v0.9) | 目标 (v2.0) |
| --- | --- | --- |
| **世界尺寸** | 120 × 160 单位 (≈ 1/100 实际) | 1000 × 1000 单位 (≈ 1/10 实际) |
| **结构** | 23 个 ad-hoc 拼贴热点 | **108 里坊 + 2 市 + 3 宫城 + 4 城门 + 朱雀大街中轴** |
| **可扩展性** | 全部硬编码在 `scene.js` (≈15k 行) | **数据驱动 manifest** (`wards/*.json`) + 模块化 hero zones |
| **历史还原度** | 名称对，地理对不上 | **依《长安志》《唐两京城坊考》定位**，可点击查阅原典 |
| **文旅价值** | 游戏体验 | **教育 + 文旅 + 学术参考**，可对接文旅局/博物馆 |

核心思路：**网格化 (Grid)** + **分层 LOD (3-tier)** + **数据驱动 (Manifest)** — 一次架构搬迁，让此后每一个新坊、新店、新 NPC 都成为一行 JSON 而非 200 行硬编码。

## 1. 项目愿景

### 1.1 目标用户

| 用户类型 | 期待价值 |
| --- | --- |
| 普通玩家 | "走在朱雀大街上，左手东市右手西市，远眺大明宫含元殿" 的沉浸感 |
| 历史爱好者 | 各坊位置、街道布局、建筑形制经得起《唐两京城坊考》比对 |
| 文旅局 / 博物馆 | 可作为陕西 / 西安文旅宣传的数字孪生底座，能够定向展示丝路、科举、佛教等主题 |
| 学校教育 | 语文/历史课堂教学情境素材 (李白游平康、玄奘归大慈恩、白居易居昭国坊) |
| 开发者 | 后续添加任意场景 / NPC / 剧情都是数据，不是代码 |

### 1.2 与现有产品的差异化定位

> 不是又一个"古风塞尔达"，也不是"皮肤化的西安城旅游 App"。

定位：**有据可考的、可对话的、可扩展的、可文旅化的唐代数字孪生平台。**

三个核心差异：

1. **历史考据驱动** — 每个坊、每条街、每座寺都附原典出处 (《长安志》《唐两京城坊考》《元和郡县图志》《大唐西域记》《入唐求法巡礼行记》)。
2. **语音 AI 原生** — Agora Voice 不是装饰，而是讲解、对话、答疑、互动游戏的核心引擎。导览者 / 历史人物 / NPC 都通过实时语音呼吸。
3. **数据 / 代码分离** — 城市本体是数据 (JSON manifest)，渲染器是代码。任何人 (策划 / 历史顾问 / 学生) 都能贡献新坊新人物，不需要会写 Three.js。

## 4. 三层 LOD 设计

| 层级 | 名称 | 数量 | 内容 | 用户体验 |
| --- | --- | --- | --- | --- |
| **L0** | **Hero Ward** | **25** (12 hero 坊 + 2 市 + 3 宫城 + 4 城门 + 1 风景区 + 3 边角) | 完整 3D，含 NPC、剧情、语音、迷你游戏、画廊、原典 | 玩家驻足游玩的主区 |
| **L1** | **Generative Ward** | **96** | 程序化生成：坊墙 + 4 门 + 十字大街 + 4 ~ 9 块民居方阵 + 1 标志 (寺/府/作坊) + 路上行人 (低密度) | 漫步穿行，感受规模 |
| **L2** | **Skybox Ward** | 远郊 / 边界 | 仅剪影 + 简单瓦顶纹理 + 远处晨钟暮鼓音效 | 视觉地平线 |

LOD 切换由 **相机距离**驱动：

| 距离视点 | 显示等级 |
| --- | --- |
| 0 – 80 u  | L0 全细节 + 全 NPC |
| 80 – 200 u | L0 几何 + L1 程序坊 全部 |
| 200 – 500 u | L1 简化 (无 NPC) + 远处 L2 |
| 500+ u  | L2 + 雾化 |

## 5. 关键场地详细设计

### 5.1 D1 宫城群 (Hero 区)

#### 5.1.1 大明宫 (主舞台)

完全按《唐代长安大明宫复原》图志：

   │      玄武门               │

   │                           │
   │  ┌─────┐               │
   │  │麟德殿│  ← 宴飨、外交  (3 殿连建)
   │  └─────┘                  │
   │                           │
   │  ┌─────┐               │
   │  │紫宸殿│  ← 内朝       │
   │  └─────┘                  │
   │                           │
   │  ┌─────┐                  │
   │  │含元殿│  ← 大朝, 龙尾道  │  L0 主入口建筑
   │  └──┬──┘                  │
   │   龙尾道                  │
   │     │                     │
   │   丹凤门 (面向太极宫)     │
   └──────┴────────────────────┘
        S ↓
```

NPC / 剧情：
- 含元殿外: **百官朝会**编队 + 万国使臣 (吐蕃、日本遣唐使、新罗、突厥)
- 麟德殿内: **李白献清平调** 触发点
- 紫宸殿: **玄宗 / 太宗 / 武则天** 三位皇帝 AI 语音 (玩家可选时代)
- 太液池: **画舫游** + 月夜钓鱼互动

#### 5.1.2 太极宫 (隋宫旧)

L0 但精简一档，重点：太极殿、两仪殿、甘露殿。NPC: **隋朝旧宫人** (与大明宫的时代差异讲解)。

#### 5.1.3 皇城官署区

L1 中略加细节的中间档，多个官署 (中书省、门下省、尚书省、御史台、鸿胪寺、太医署、司天监) 作为可入门牌坊群。

    │   │   │   │   │        │
    │ [行] [行] [行] [行]     │
    │   │   │   │   │        │
    │ [行] [行] [行] [行]     │
    └─────────────────────────┘
```

**西市**专设 (与现状西市 / 染坊整合并扩充):
- 波斯邸 (酒肆 + 胡姬 NPC)
- 大秦景教碑场景 (附碑文原文 + 语音讲解)

## 6. 文旅价值 — "数字孪生文旅平台" 三层

### 6.1 教育层 (Edu Layer)

每个 L0 hero region 配 **原典 + 注释 + 语音讲解**:
- 原典页 (HTML 内嵌《长安志》《唐两京城坊考》对应条目)
- 注释页 (考古/史学界主流注释)
- 语音讲解 (Agora Voice, 1-3 min 标准导览 + 自由问答)

### 6.2 互动层 (Interactive Layer)

- 已有: 街头小游戏 (蹴鞠 / 雁塔题名 / 曲江流觞 / 弓射 / 摇骰子 / 划拳 / 斗草 / 猜枚 / 饮酒对诗)
- 新增 (Phase 4 后):
  - **科举闯关** (国子监 → 进士及第 → 曲江宴): 串联 D2 + D8
  - **丝路商队** (西市 → 西出阳关 → 敦煌 → 回长安): 跨场景任务
  - **玄奘西行** (大慈恩寺出发, 看玄奘壁画后触发): 串联 D6 + 远郊
  - **白居易一日** (跟随白居易上朝 → 翰林院 → 昭国坊家): 串联 D1 + D7
  - **节庆系统**: 元日大朝、上巳修禊、清明寒食、中秋玩月、重阳登高 (按真实日期触发场景模式)

### 6.3 展示层 (Exhibit Layer) — 文旅局可用

- **导览模式**: 一键 30 分钟全城精华导览 (剧本 + 自动镜头 + Agora 解说)
- **快照导出**: 玩家在任意场景可一键导出 8K 截图 + 带历史注释 (用于教学/分享)
- **数据导出**: 可向博物馆 / 文旅局开放 manifest 数据 API，他们可接入自有 App / Web
- **二次创作**: 开放 ward JSON 编辑器，文化工作者无需编程即可贡献新坊

## 10. 风险与备选

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 重构期现有功能 (语音、画廊、迷你游戏) 中断 | 高 | Phase 1 用 **strangler-fig 模式**：保留 `scene.js` 旧入口，新框架先并行跑，逐 zone 迁移；每迁移完一个跑一遍 e2e |
| 1000×1000 单位渲染性能 | 高 | InstancedMesh、frustum culling、distance LOD、贴图共享、按 chunk 分批 add 到 scene |
| 108 ward 数据创作工作量 | 中 | L1 模板 + 12 个组件 (寺/府/作坊/民居/酒肆/官署) 随机搭配, 每坊 manifest ~20 行 JSON |
| 文化考据准确度 | 中 | Phase 0 输出 **考据来源清单**, Phase 3-4 每加新场景必须列史料原文 |
| 语音 (Agora) 跨大场景延迟 | 低 | 现有 ConvoAI 架构已 ok, 但多 NPC 同屏需测试; 必要时引入 NPC 距离 mute 策略 |

## 11. 下一步决策点 (需用户回答)

1. **世界尺寸**: 1000×1000 u (1:10) 还是 700×700 u (1:14, 更易渲染但小一些)?
