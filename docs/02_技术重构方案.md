## 2. 坐标系约定 (Coordinate Convention)

### 2.1 世界尺度

- **1 unit = 10 m** (实际尺寸 ÷ 10)
- 长安外郭城: **970 u (E-W) × 865 u (N-S)** ≈ 推荐世界范围 1000 × 1000
- 朱雀大街中线: `x = 0`, 即 `x < 0` 为西长安县, `x > 0` 为东万年县
- 北墙 (含元殿一线): `z = -432`
- 南墙 (明德门一线): `z = +432`
- y 轴向上, 地面 `y = 0`

### 2.2 网格 (Ward Grid)

- 行 (row): N→S, 编号 0 (最北一排) ~ 12 (最南一排)
- 列 (col): W→E, 编号 0 (最西一列) ~ 10 (最东一列)
- 朱雀大街切在第 5 列与第 6 列之间 (即 col 0-4 = 西县, col 6-10 = 东县, col 5 留给朱雀大街)
- 每坊默认 60 × 60 u (含坊墙), 街道 8 u (一般) / 16 u (朱雀)

```js
// lib/world/grid.js
export const GRID = {
  rows: 13,
  cols: 11,
  wardSize: 60,                // 每坊 60 × 60 u
  streetW: 8,                  // 一般街道
  zhuqueW: 16,                 // 朱雀大街
  centerCol: 5,                // 朱雀所在列
  worldOrigin: { x: -485, z: -432 }, // 城西北角
};

export function wardToWorld(row, col) {
  // 返回该坊中心点的世界坐标 {x, z}
  // ...
}

export function worldToWard(x, z) {
  // 反查该点所在坊 {row, col} 或 null (在街道上)
}
```

### 2.3 ID 命名规范

- L0 hero 坊: `ward-<拼音>` (例 `ward-pingkang`, `ward-jinchang`)
- L0 非坊 (宫城、市、城门): `region-<拼音>` (例 `region-daming`, `region-eastmarket`)
- L1 程序坊: `ward-r<row>c<col>` (例 `ward-r02c08` = 北 2 排东 8 列)
- L2 远郊: `bg-<方位>` (例 `bg-north`)

## 3. 三层 LOD 架构

```
┌─────────────────────────────────────────────────┐
│                  Camera                          │
│        ↓ distance + frustum                      │
│        ↓                                         │
│  ┌──────────────────┐    ┌──────────────────┐  │
│  │  ChunkLoader     │ →  │  WardRegistry    │  │
│  │  (按距离触发)     │    │  (manifest+impl) │  │
│  └────┬─────────────┘    └────┬─────────────┘  │
│       │ load                  │ query           │
│       ↓                       ↓                  │
│  ┌──────────────────────────────────────────┐  │
│  │  Three.js Scene (按 LOD group)            │  │
│  │  ├ L0 group  (always visible if active) │  │
│  │  ├ L1 group  (instanced, frustum culled)│  │
│  │  └ L2 group  (skybox, simple)           │  │
│  └──────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

### 3.1 LOD 切换策略

```js
// lib/world/lod.js
export const LOD_THRESHOLDS = {
  L0_full: 80,      // 0-80u: full geometry + NPC
  L1_visible: 200,  // 80-200u: L0 keep + L1 visible
  L1_simplified: 500, // 200-500u: L1 reduce (no NPC)
  L2_only: Infinity, // 500+: skybox
};

export class LODManager {
  constructor(scene, camera) { /* ... */ }

  update(cameraPos) {
    // 每帧 (节流到 0.5s) 重新评估所有 region 的 LOD
    for (const region of this.regions) {
      const d = region.center.distanceTo(cameraPos);
      const newLOD = this.pickLOD(d, region);
      if (newLOD !== region.currentLOD) {
        this.switchLOD(region, newLOD);
      }
    }
  }
}
```

### 3.2 性能预算

| 层级 | 最大同屏数 | 顶点上限 | NPC 上限 | InstancedMesh 必用 |
| --- | --- | --- | --- | --- |
| L0 | 4-6 (用户附近 + 视野内) | 200k 总计 | 30 个 | 否 (允许精细) |
| L1 | 30-50 | 80k 总计 | 0 (远) | 是 |
| L2 | 全部远郊 | 5k | 0 | 是 |

## 7. 性能策略详细

### 7.1 必用技术

- **InstancedMesh** for 同类建筑 (民居 / 树 / 灯笼)
- **Frustum culling** 自动 (Three.js 默认开)
- **Distance culling** (手动): chunk-loader 控制
- **Geometry sharing**: 同形屋顶 / 柱础 / 瓦当只创建一次, instance 多次
- **Texture atlasing**: 把多套小贴图打包成大图, 减少 draw call
- **LOD geometry**: 同一建筑准备 high/mid/low 三档 mesh (BufferGeometry simplify)
- **Layer-based culling**: NPC / 装饰 / 主体建筑分 layer, 远处可一键关 NPC layer

### 7.2 性能预算

| 指标 | 目标 |
| --- | --- |
| draw calls | < 1000 |
| 顶点 | < 800k |
| 同屏 NPC | < 40 |
| 单帧 update 时长 | < 8 ms |
| GC pressure | 无 per-frame new (复用 Vector3) |

### 7.3 性能监控

引入 `stats.js` (开发时), 暴露 URL `?debug=perf` 显示帧时、draw call、triangle 数。

## 9. 测试与 debug hooks

## 11. 验收标准 (Definition of Done)
