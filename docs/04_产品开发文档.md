## 1. 产品总览

### 1.1 一句话简介

> **「在浏览器里走进盛唐长安，按住一句话，和李白、画学博士、AI 品牌智机使实时对话。」**

一个基于 Three.js 的低多边形大唐长安沙盒文旅游戏，**深度集成 Agora ConvoAI**，把"Agora 实时音视频 + AI 大模型"装进每一个 NPC，让玩家用自然语言驱动游戏：寻路、看画、对诗、品牌问答、上传作品。

### 1.2 产品定位

| 维度 | 定位 |
|---|---|
| 表层 | 一款 web 端、轻量、可分享的"古风文旅 + 角色扮演"游戏 |
| 中层 | 一份"Agora ConvoAI · 全要素技术展演"，覆盖中文低延迟语音、多 Persona、context bridge、intent routing、TTS 音色矩阵 |
| 底层 | 一个**古今穿越**叙事容器：长安城里出现戴 Vision Pro 的老学究、自拍胡商、奶茶仕女，"AI 一条街"用 9 个唐风殿宇代表当代主流 AI 品牌 |

### 1.3 三大体验支柱

1. **实时语音 NPC**
   1 秒级 RTT 的人机对话，覆盖 18 个 persona（5 个诗人 / 1 个导游 / 1 个守门将 / 1 个画学博士 / 9 个 AI 品牌智机使 / 1 个 Ada）。
2. **古今穿越文旅**
   主城是唐风长安低多边形 1200 m² 沙盒；插入"AI 一条街 · 天枢府"独立坊作为现代时空裂口。
3. **小游戏 + UGC**
   猜拳 / 飞花令 / 解灯谜 / 染色配方 / 弓射 / 蹴鞠 / 望闻问切等 22+ 类小游戏；丹青馆支持照片上传 / AI 文生图 / 题款挂墙。

### 1.4 目标用户

- **B 端**：Agora 客户、ConvoAI 评估方、文旅局、博物馆数字化、教育出版社
- **C 端**：泛文化爱好者、汉服圈、AI 早期玩家、家长带孩子做"沉浸式古文教学"

## 2. 核心特色 · 实时语音 ★

> 这一节是整个项目的灵魂；其它功能都是它的载体。

### 2.1 实时无延迟对话

#### 2.1.1 玩家可感知体感

```
[玩家按住说话] ─── 鼠标点 NPC ─── 唤起 Agora 面板
       ↓
[ ~1.5–2.5 秒 ]   ◀── 包含 RTC 接通 + STT + LLM + TTS + 首包到耳
       ↓
[ NPC 用专属音色开口回话, 头顶飘字幕, 左侧视频角色 talking 动画 ]
```

竞品体感延迟普遍 6–7 秒（含本地 ASR、网络往返、独立 LLM 调用），本产品借 Agora ConvoAI 的 **VAD + 端到端管道融合**，把这个数字压到 1–2 秒级，**单轮 RTT 大约是『从你说完到你听到 NPC 第一个音节』的时间**，而不是『从你说完到 NPC 说完』。

#### 2.1.2 端到端链路与延迟拆解

| 阶段 | 平均耗时 | 实现位置 | 优化点 |
|---|---|---|---|
| ① 麦克风 → 浏览器 RTC 上行 | < 50 ms | `agora-rtc-sdk-ng` `useLocalMicrophoneTrack` | `ENABLE_AUDIO_PTS = true` 时间戳同步 |
| ② RTC 到达 Agora 边缘节点 | < 30 ms | Agora SD-RTN™ 全球加速 | 走对应区域接入 |
| ③ VAD 判断"说完" | 800 ms 静默阈值 | `agent.py` VAD `silence_duration_ms=800` | 平衡"打断 vs 误触" |
| ④ STT（流式） | 200–500 ms | Ares STT (中文) / Deepgram (英文) | 流式增量，不等整段 |
| ⑤ LLM 首 token | 300–700 ms | OpenAI gpt-4o-mini 管道内 | `max_tokens=240` + 100 字 prompt 约束 |
| ⑥ LLM 流式输出到 TTS | 与 ⑤ 并行 | ConvoAI 内部 streaming | 边出边读，不等全句 |
| ⑦ MiniMax TTS 首音 | 300–600 ms | `speech_2_6_turbo` | "turbo"档专为低延迟 |
| ⑧ TTS 音频 → RTC 下行 → 玩家 | < 80 ms | RTC 端到端 | — |
| **首字延迟（⑨ 总和）** | **~1.5–2.5 s** | | |

**关键优化**（在我们的代码层做的，不是 SDK 默认行为）：

1. **Prompt 字数硬约束**：每个 persona 的 `system_prompt` 都强制"30–130 字回答"。短回复 = 更快说完 = 玩家更快听到结尾。
2. **`max_tokens=240`**：硬上限，再啰嗦也不让它跑长。
3. **VAD 调参**：`silence_duration_ms=800ms` 让玩家自然停顿不被切断，`interrupt_duration_ms=500ms` 允许玩家打断 NPC，`prefix_padding_ms=400ms` 防开头吞字。
4. **业务层 debounce**：场景上下文（玩家切视角 / 走到新坊）合并到一个 5–6 秒窗口注入，不让 LLM 被微抖动事件淹没。
5. **音量自动 duck**：玩家打开语音面板，BGM 音量自动降到 12%，麦克风信噪比立刻上去，STT 错误率下降。
6. **Persona 暖机**：iframe 加载后 5 秒内不发 context，避免 sendText 先于 agent ready 而丢消息。
7. **同 persona 不重建**：玩家短时间多次点同一 NPC，iframe 不重新 join channel，省 1 次 RTC 握手。
8. **专属音色**：18 个 persona 各自的 `tts_voice_id`，避免"千人一声"，强化角色记忆。

#### 2.2.2 完整数据流（点 NPC → 听到回答）

```
1. Player clicks NPC sprite in han-diorama scene
    └─ scene.js openVoicePanel(npc)
       · iframe.src = `${VOICE_AGENT_ORIGIN}/?persona=libai&autoStart=1&embed=1`
       · openPortraitPanel('libai') ← 左侧角色形象面板出场
       · beginDialogueFraming(npc) ← 镜头切换 + NPC 转身

2. iframe boots (Next.js page) → useAgoraConnection.connect({ personaId: 'libai' })
    a. GET /api/get_config            → FastAPI 生成 Token007 (RTC + RTM 共用)
    b. RTC useJoin(channel, uid, token)
    c. useLocalMicrophoneTrack() + usePublish()  ← 玩家麦克风上行
    d. RTM login(uid, token) + subscribeChannel  ← 文字流双向通道
    e. AgoraVoiceAI.init({ rtcEngine, rtmEngine })
    f. POST /api/v2/startAgent { channelName, personaId: 'libai', uids }
        → FastAPI agent.py Agent.start(persona)
            · get_persona('libai') ← 动态 import personas/libai.py
            · build vendor chain: AresSTT zh-CN → OpenAI gpt-4o-mini → MiniMaxTTS Lyrical_Voice
            · ConvoAI 创建云端 agent, join 同一个 channel
        → 返回 { agentId }

3. Player 说话
    Mic → RTC → Agora Edge → ConvoAI VAD → STT → LLM → TTS → Agent Audio Stream → RTC → Player Speaker
                                                              │
                                                              └─▶ RTM TRANSCRIPT_UPDATED
                                                                  └─▶ iframe app.tsx postMessage
                                                                      { source: 'tang-voice-agent',
                                                                        type: 'transcript',
                                                                        personaId, isAgent: true, text, status }
                                                                      ▼
                                                          han-diorama scene.js message listener
                                                              · showOverheadSubtitle('libai', text)
                                                                ← NPC 头顶飘唐风字幕

4. AGENT_STATE_CHANGED: 'speaking' → postMessage { type: 'state', state: 'talking' }
    ← han-diorama setPortraitTalking(true)
      · 左侧 portrait panel 切到 talking.mp4
      · GLB 角色播 talk 动画
      · BGM duck 到 12%

5. Player 用户主动关闭 (Esc 或 X)
    → iframe POST /api/v2/stopAgent
    → ConvoAI session.stop(); RTM logout; track cleanup
    → postMessage { type: 'closed' }
    → han-diorama closeVoicePanel()
      · iframe.src = 'about:blank' ← 销毁子页面回收 RTC
      · closePortraitPanel()
      · endDialogueFraming() ← 还原镜头
      · BGM 音量恢复
```

#### 2.2.3 iframe ↔ 主场景 双向消息桥

**主场景 → iframe**：通过 `postMessage` 注入"场景上下文"，让 NPC 知道玩家在哪、刚做了什么。

```javascript
// han-diorama emitPersonaContext()
frame.contentWindow.postMessage({
  source: 'tang-host',
  type: 'context',
  kind: 'stage' | 'interrupt',     // stage = 排队追加, interrupt = 打断 barge-in
  contextKind: 'tour_guide:view',  // 用作 debounce key
  text: '[场景提示] 玩家切到了俯瞰视角，请用 15 字点评。',
}, VOICE_AGENT_ORIGIN);
```

iframe 侧 `app.tsx` 收到后调 `AgoraVoiceAI.sendText(text, ChatMessagePriority.APPEND|INTERRUPTED)`，注入到当前对话上下文。

**iframe → 主场景**：iframe 把"AI 状态变化"和"用户语音 intent"广播给主场景。

| `type` | 时机 | payload 关键字段 |
|---|---|---|
| `state` | visualizer 状态变化 | `state: 'joining' \| 'listening' \| 'analyzing' \| 'talking' \| 'ambient' \| 'disconnected'`, `personaId` |
| `transcript` | RTM 文本流增量 | `personaId, turn, isAgent, text, status: 'partial' \| 'final'` |
| `intent` | 用户说完一句被 parseIntent 命中 | `kind: 'walkTo' \| 'findNpc' \| 'setView'`, `payload: { place, npc, mode }` |
| `closed` | 玩家点 iframe 内"退场" | `{ personaId }` |

详细协议见 [附录 A](#附录-a-iframe-postmessage-协议)。

#### 2.2.4 Persona 系统设计

后端 `tang-voice-agent/server/src/personas/` 下每个文件一个 persona：

```python
# personas/libai.py
PERSONA = Persona(
    persona_id="libai",
    display_name="李太白",
    role="诗仙 · 飞花令对诗",
    language="zh-CN",
    stt_vendor="ares",
    stt_language="zh-CN",
    tts_voice_id="Chinese (Mandarin)_Lyrical_Voice",
    tts_model="speech_2_6_turbo",
    llm_model="gpt-4o-mini",
    system_prompt="""...你是李白...30-60 字回答...遇含"风/花/月"等字主动接飞花令...""",
    greeting="同游长安否? 一壶浊酒, 喜逢知己。",
)
```

**`?persona=xxx` URL 路由**：iframe 启动时从 query 读 `personaId`，传给 `connect()` → `POST /v2/startAgent { personaId }` → 后端 `get_persona(id)` → 动态 import 拿到 PERSONA。

**`make_brand_persona()` 工厂**：9 个 AI 品牌智机使共享一份"AI 一条街讲解员"基底 prompt + 知识库，每个馆只填差异 4 行（`display_name`、`tts_voice_id`、`greeting`、`extra_prompt`）。维护成本极低。

### 2.4 三维场景 ↔ 语音对话联动

| 联动场景 | 实现 |
|---|---|
| 玩家走到 NPC 前自动转身 | `beginDialogueFraming(npc)` 保存原朝向 → NPC `rotation.y = atan2(player − npc)` |
| 进入语音对话切镜头 | TPS 状态下保存当前 `cameraMode`，对话结束 `endDialogueFraming()` 还原 |
| AI 说话时 NPC 摆手 | iframe `state: 'talking'` → `window.voiceAiSpeaking = true` → animate loop 把 GLB 切到 `talk` clip |

| 玩家说"带我去大雁塔" | iframe `parseIntent('walkTo', { place: '大雁塔' })` → postMessage → `walkPlayerTo('大雁塔')` → 主角自动 WASD 寻路 |

- 打开面板 → idle.mp4 循环 + 触发 intro.mp4 有声开场

### 5.1 进入游戏到自动导览

```
1. 玩家访问 http://localhost:8126/index.html
2. 点右上角"走进长安"CTA → 弹出角色选择卡

5. 2.5 秒后自动 openAmbientTourGuide():
   · 右侧滑出 voice panel, iframe 加载 ?persona=tour_guide&autoStart=1&embed=1
   · 左侧 portraitPanel 同步出场, 显示周引之 intro 视频（有声）

```

## 6. 性能、可靠性、安全

### 6.1 延迟与抖动

| 维度 | 措施 |
|---|---|
| Agora 边缘节点 | 走 SD-RTN™，国内 < 30 ms RTT |
| 同 persona 复用 iframe | 玩家短时间点同一 NPC 不重建 RTC channel，省 1 次握手 |
| Token 续约 | `token-privilege-will-expire` 自动刷新 RTC + RTM token（1 小时） |
| Context debounce | 同一 persona 同一 kind 至少间隔 5–6 秒，避免 sendText 风暴 |
| 麦克风优先级 | 进入语音对话 BGM 自动 duck 到 12%，提高 STT 信噪比 |
| 视频肖像 lazy | 切 persona 才 load 对应 idle/talking，不 preload 全套 |
| postFX | 默认 OutlinePass + 弱 Bloom（emissive 驱动），CPU 占用低 |

### 6.2 失败兜底

| 失败模式 | 兜底 |
|---|---|
| 麦克风权限拒绝 | iframe 显示提示，不挂死 |
| Agora 连接失败 | iframe 显示 error bar，不会污染主场景 |
| iframe 漏发 talking-end | scene.js `_voiceTalkSafetyOff` 30 秒兜底关 talk 动画 |
| `talking.mp4` 不存在 | 不切，保持 idle 循环（`talk.readyState >= 2` 判断） |
| `intro.mp4` autoplay 被浏览器拦 | toast 提示 "点击人物后可播放"，不影响后续对话 |
| 自定义品牌 logo 加载失败 | 自动 fallback 到品牌 Latin 大字纹理 |
| BGM 加载失败 | 自动跳过当前曲，切下一首 |
| Cloudflare 拦截下载 | 文档明示 archive.org 来源，README 给出替换路径 |
| 玩家关闭 voice panel | iframe.src = 'about:blank' 销毁子页面，回收 RTC 资源 |
| 同 NPC 二次点 | iframe 不重建，加 flash 高亮提示玩家已在对话 |
| postMessage 错域 | 检查 `data.source === 'tang-voice-agent'`，过滤无关消息 |

### 6.3 资源加载策略

- Three.js 用 import map 走 unpkg CDN，无需 bundler
- GLB 走 Google DRACO CDN 解码
- 肖像视频 ≈ 2–3 MB / persona，按需 load
- BGM 96 kbps mono，每曲 2–3 MB
- 主项目静态资源走本地 :8126 简单 http.server

### 6.4 安全

| 面 | 措施 |
|---|---|
| Agora 凭据 | 仅放 `server/.env.local`，前端只拿临时 Token007（1h 过期） |
| CORS | 后端 `*`，仅本地开发；生产应限定 origin |
| postMessage 域 | 主场景发送时指定 `VOICE_AGENT_ORIGIN`，接收时验 source 字符串 |
| 自定义品牌 | 用户上传内容仅存 `localStorage`，不上传服务器 |
| UGC（丹青馆） | 同上，全部本地存储 |
| 麦克风权限 | 浏览器原生提示，不绕过 |

### 7.4 换头像视频

- `intro` 带音频用于开场
- `idle` 必须无声（用 `ffmpeg -an`）
- `talking` 无声 + 嘴动

### 7.5 换 BGM

把 `.mp3` 放到 `assets/bgm/`，在 `scene.js` 的 `bootTangBGM` 内 `PLAYLIST` 加一行：

```javascript
{ src: 'assets/bgm/your-track.mp3', title: '显示标题', meta: '副标 / 作者' }
```

## 9. 附录

### 附录 A · iframe postMessage 协议

```typescript
// ───────────── 主场景 → iframe ─────────────
interface HostToFrame {
  source: 'tang-host';
  type: 'context';
  text: string;
  kind?: 'stage' | 'interrupt';     // stage 排队追加 (默认), interrupt 立即打断
  contextKind?: string;             // 用于 debounce key, e.g. 'tour_guide:view'
}

// ───────────── iframe → 主场景 ─────────────
interface FrameToHost {
  source: 'tang-voice-agent';
  type: 'state' | 'transcript' | 'intent' | 'closed';
}

interface StateMessage extends FrameToHost {
  type: 'state';
  state: 'disconnected' | 'joining' | 'not-joined'
       | 'ambient' | 'listening' | 'analyzing' | 'talking';
  personaId: string;
}

interface TranscriptMessage extends FrameToHost {
  type: 'transcript';
  personaId: string;
  turn: number;
  uid: string;
  agentUid: string;
  isAgent: boolean;
  text: string;
  status: 'start' | 'partial' | 'end' | 'final';
}

interface IntentMessage extends FrameToHost {
  type: 'intent';
  kind: 'setView' | 'walkTo' | 'findNpc';
  payload:
    | { mode: 'isometric' | 'follow' | 'topdown' | 'cinema' | 'fps' }
    | { place: string }
    | { npc: string };
  personaId: string;
}

interface ClosedMessage extends FrameToHost {
  type: 'closed';
  personaId: string;
}
```
